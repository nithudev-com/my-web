# Next Ecommerce ISR Starter

Basic ecommerce starter for a speed-first Next.js store with:

- Next.js App Router
- Product/category ISR pages
- PostgreSQL + Prisma
- Redis + BullMQ product import queue
- Meilisearch product search sync placeholder
- CSV product importer
- On-demand revalidation API
- Split product sitemap support
- Product JSON-LD schema
- Basic admin import screen

## Architecture

```txt
Visitor
  ↓
CDN / Next.js cache
  ↓
ISR product/category pages
  ↓
PostgreSQL for product/order data
  ↓
Redis cache + BullMQ importer
  ↓
Meilisearch for search/filter
```

## Setup

```bash
cp .env.example .env
docker compose up -d
npm install
npm run prisma:migrate
npm run seed
npm run dev
```

Open:

```txt
http://localhost:3000
http://localhost:3000/category/demo-category
http://localhost:3000/admin/import
```

## Import products

Use the demo CSV:

```txt
public/sample-products.csv
```

For small test files, the API can process immediately. For 2 lakh products, run the worker:

```bash
npm run worker:import
```

## CSV columns

```txt
sku,title,slug,price,salePrice,stockQuantity,categoryName,categorySlug,brandName,brandSlug,imageUrl,seoTitle,seoDescription
```

Required:

```txt
sku,title,price
```

## ISR strategy

Public SEO pages:

```txt
/                 revalidate 15 min
/product/[slug]   revalidate 60 min + tag product:slug
/category/[slug]  revalidate 30 min + tag category:slug
/sitemap.xml      generated by Next.js
/product/sitemap/[id].xml split product sitemap
```

Private/dynamic pages:

```txt
/admin
/cart
/checkout
/account
/api
```

## Revalidate API

POST:

```bash
curl -X POST http://localhost:3000/api/revalidate \
  -H "content-type: application/json" \
  -d '{"secret":"change-this-secret","paths":["/","/product/demo-product-1"],"tags":["products","product:demo-product-1"]}'
```

## 2 lakh product rule

Do not generate every product at build time.

The product page uses:

```ts
export async function generateStaticParams() {
  return getTopProductSlugs(1000);
}

export const dynamicParams = true;
export const revalidate = 3600;
```

That means:

- Top products are prebuilt.
- Other products are generated when first visited.
- Updated products are refreshed through ISR revalidation.
